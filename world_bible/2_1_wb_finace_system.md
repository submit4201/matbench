## Costom addtions 

┌─────────────────────────────────────────────────────────────────────────┐
│                         CUSTOM PROPOSAL SYSTEM                          │
├─────────────────────────────────────────────────────────────────────────┤
│ PURPOSE                                                                 │
│ • Allows human players or AI agents to invent new revenue streams       │
│   beyond predefined options.                                            │
│ • Ensures creativity is balanced with feasibility and economic realism. │
│                                                                         │
│ STRUCTURE                                                               │
│ 1. Proposal Submission                                                  │
│    • Player/AI submits idea via Ticketing/Comms Channel.                │
│    • Required fields:                                                   │
│       – Service Name                                                    │
│       – Category (Wash, Dry, Bundle, Premium, Ancillary, Other)         │
│       – Description                                                     │
│       – Pricing Model (per load, per item, subscription, etc.)          │
│       – Resource Requirements (equipment, staff, costs)                 │
│                                                                         │
│ 2. LLM Evaluation                                                       │
│    • System analyzes proposal against game state.                       │
│    • Outputs structured JSON with:                                      │
│       – Feasibility Score                                               │
│       – Profitability Estimate                                          │
│       – Customer Appeal Assessment                                      │
│       – Upkeep Steps (inventory, staffing, maintenance)                 │
│                                                                         │
│ 3. Decision Options                                                     │
│    • Approve → Adds service to active revenue streams.                  │
│    • Reject → Discards idea, no changes made.                           │
│    • Defer  → Stores idea in “Opportunities Board” for later review.    │
│                                                                         │
│ 4. Implementation & Tracking                                            │
│    • Approved services appear in Management System.                     │
│    • NPCs/customers react with feedback (positive/negative/neutral).    │
│    • Service tracked like predefined streams (revenue, costs, demand).  │
│                                                                         │
│ EXAMPLES                                                                │
│ • Overnight Laundry Lockers → Requires lockers, maintenance vendor.     │
│ • Eco-Friendly Subscription → Needs green detergent supplier, staff.    │
│ • Community Workshops → Requires staff time, boosts reputation.         │
└─────────────────────────────────────────────────────────────────────────┘
